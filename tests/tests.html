Golden

What is Golden?

Golden is a CSS framework built using the Golden ratio. It can be used independently, or incorporated into other CSS frameworks like Twitter Bootstrap.

The Golden ratio has been found in an uncoincidentally large number of applications in nature and math. As a result of it's apparent significance and mathematical properties, it has been used by humans in art and design for thousands of years. Learn more about the Golden ratio here and about how it relates to modern website design here.

Getting Started

1. Download Golden

Download Golden here or check out Golden on Github.

2. Include Golden

Include Golden in your project. You can copy the basic template below to get started fast.

Example

Hello, world!

<!DOCTYPE html>
<html>
  <head>
    <title>Golden Starter Template</title>
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- Golden -->
    <link href="css/golden.min.css" rel="stylesheet">
  </head>
  <body class="golden">
    <h1>Hello, world!</h1>
  </body>
</html>

Everything in Golden is rooted under the .golden class. This makes it easy to apply Golden selectively to certain sections in your project, or globally like in the example above.

4. Understand Golden

Golden was built to extend naturally from Twitter Bootstrap while still being able to stand alone as an independent framework. It has borrowed heavily from Bootstrap's philosophy and is made in a similar architectural style. It is designed to be both quick and extensible, modular and streamlined, customizable and simple.

At the outer layer is a pre-compiled copy of Golden, golden.css, a pre-compiled minified copy of Golden, golden.min.css, and the default Golden, golden.less, from which the first two are compiled from.

golden.less

              // Bootstrap stuff
              // Include helpful Bootstrap modules to make life easier later on.

              // Include the Bootstrap variable defaults
              // These can be edited at will, or replaced with a custom copy of Bootstrap variable overrides
              @import "./bootstrap/variables.less";

              // Include the groundwork Bootstrap mixins and classes for conveniance
              @import "./bootstrap/mixins.less";
              @import "./bootstrap/utilities.less";

              // For cross-browser consistency see git.io/normalize
              @import "./bootstrap/normalize.less";

              // For printing
              @import "./bootstrap/print.less";

              // Include the basic Bootstrap 12 column grid for convensiance
              @import "./bootstrap/grid.less";

              // Golden stuff
              // Let's make life Golden :)

              // Override the Bootstrap variables with values based on the golden-ratio
              @import "./golden/variables.less";

              // Golden modules that require the Golden variables
              // These do not affect any pre-existing Bootstrap CSS and are thus not rooted under the .golden class
              @import "./golden/mixins.less";
              @import "./golden/grid.less";
              @import "./golden/content.less";

              // Root the Golden styles that will affect pre-existing Bootstrap CSS under the .golden class
              .golden {
                @import "./golden-bootstrap/golden-bootstrap.less";
              }

              // Add a special class for only including the Golden typography styles
              .golden-typography {
                @import "./golden-typography/scaffolding.less";
                @import "./golden-typography/typography.less";
              }

The variable dependencies in Golden are designed to maximize ease of customization and expansion, however they can be confusing at first. Golden expects, and requires, that a base set of variables for the project be defined in traditional Bootstrap style. These can be customized by you, or just left as the Bootstrap defaults as found in boostrap/variables.less that are used in the pre-compiled Golden.

Golden then takes the Bootstrap variables and applies the golden ratio to create a new set of Golden variables in golden/variables.less. These variables are used by the Golden modules to create a grid system and more.

Finally, Golden takes the Golden variables and uses them to override the Bootstrap defaults in golden-bootstrap/variables.less. These variables are used by the Bootstrap inspired modules in golden-bootstrap/golden-bootstrap.less to create a custom version of Bootstrap based on the golden ratio.

golden-bootstrap.less

              // Override Bootstrap variables with Golden values
              @import "./variables.less";

              // Include Bootstrap modules
              @import "./scaffolding.less";
              // ...

In order to make this possible, Golden roots all of it's Bootstrap override code under the .golden class. This has the added benefit of preventing Golden from conflicting with any pre-existing styles, and giving you complete control over the extent to which Golden is applied.

5. Hack Golden

Since Golden is cleanly modularized, and extends pre-existing Bootstrap modules when possible by overriding actual Bootstrap variables, it's easy to customize Golden by adding or removing Bootstrap modules, and it's easy to customize Bootstrap by adding Golden modules.

Customize Golden with Bootstrap

Golden creates its Golden styles based on Bootstrap variables defined in bootstrap/variables.less. Similar to the first step in customizing Bootstrap, the first step in customizing Golden is to alter these variables.

Golden overrides the same variables that are used by Bootstrap in golden-bootstrap/variables.less, so it's easy to add and remove Bootstrap modules. Simply take a Bootstrap module as is from Bootstrap and add it to the golden-bootstrap/ directory. Then include the modules in golden-bootstrap/golden-bootstrap.less and recompile.

Customize Bootstrap with Golden

If you have an existing Bootstrap project that you want to apply Golden styles to it's easy. Include the Golden variables and Bootstrap variable overrides before including any Bootstrap modules. Also include any Golden modules that you want. An example is provided below.

bootstrap.less

              // Core Bootstrap variables and mixins
              @import "variables.less";
              @import "mixins.less";

              // Core Golden variables
              @import "./golden/variables.less";

              // Golden modules
              @import "./golden/mixins.less";
              @import "./golden/grid.less";
              @import "./golden/content.less";

              // Golden Bootstrap variable overrides
              @import "./golden-bootstrap/variables.less";

              // Bootstrap modules
              @import "scaffolding.less";
              // ...

Documentation

Grid System golden/grid.less

Golden extends the Bootstrap grid system you're already comfortable with by adding containers and columns with proportions based on the Golden ratio. Note that the Golden grid system is fully compatible and non-destructive towards the Bootstrap grid system.

The Golden grid system follows a predictable and intuitive naming convention for classes, .golden-[element]-[screen-size]-[element-size]. Golden uses the existing Bootstrap screen sizes, xs, sm, md, and lg, to set responsive breakpoints for resizing content. To read more about the Bootstrap grid system go here.

Containers .golden-container

The Golden container is a centered div with a golden relationship between it's width and left and right margins. It also incorporates media queries to resize for different devices.

The Golden container can be customized by changing the class to something in the form of .golden-container-[screen-size]-[container-size]. On devices smaller than the given screen-size, the Golden container will extend to 100% of the available width.

The following example will create a wide Golden container on sm devices and above. On xs devices the container will have a width of 100%.

Columns .golden-col

Golden adds two new columns, wide and narrow, whose relative widths are related by the Golden ratio. The two types of columns are meant to be used together, so a wide columns and a narrow column will always add up to 100% of the available width. Similar to Bootstrap columns, the Golden columns will collapse at smaller screen sizes.

Typography golden-bootstrap/type.less

The Golden typography is derived from the @font-size-base variable in bootstrap/variables.less. By default this font size is 14px. Golden uses this value to create custom line heights and widths that are re-used in other components for margin and padding.

Due to the importance of typography, it can be accessed independently from the rest of the Golden framework through the .golden-typography class.

Much of the Golden typography was inspired from a brilliant blog post which I suggest reading if you want to understand the math in greater detail.

Additionally, the Golden typography utilizes the Golden colored gray scale. The default text color is a dark shade of gray related to both pure white and pure black by the Golden ratio.

Headings and Subheadings

Golden typography includes all HTML headings, and a small tag for creating subheadings. Each heading is related to the base font size by progressively larger powers of the Golden ratio. Each subheading is related to it's heading by the Golden ratio. Additionally, the color of each subheading is related to the color of its heading by the Golden ratio.

Paragraphs

Typographical Elements

Golden includes all the other typographical elements that Bootstrap includes, but without any additional customizations beyond what's already been discussed. These elements will, for the most part, inherit the base font size, line height, and Golden gray scale. They will also have an added margin-bottom based on the line height, to help keep adjacent elements nicely spaced out.
